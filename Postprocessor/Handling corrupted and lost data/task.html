Description

One of the common problems is that the data is incorrect. As you've seen earlier in the database.csv and the hash_database.csv files, there are lines where user information values are missing. Any part of the data from the nickname to the consent may be missing. Complete the data-cleaning process and get rid of incomplete information. To simplify the task, we consider a data gap to be a character - instead of any other value.

As a result of this step, you will get the filtered_database.csv file that contains correct data only, all passwords are converted to hashes, and the indexes are numbered correctly.

Please note that the hash_database.csv file does not need to be deleted or changed, it should remain the same as in the previous step if you successfully completed it.

Objectives

We've created the filtered_database.csv file, similarly to the previous stage, it contains the only line id, nickname, password, consent to mailing.

In this stage you should:

  • Delete the rows with incomplete data;

  • Reorder the row indexes from 1 to the last value because when deleting rows with missing information, the indexes are lost.
