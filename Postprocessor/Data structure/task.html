Description

Developers quite often make bad decisions about where and in what form they will store data. A bad decision can cause harm to a company if it does not think about the reliable storage of information. For new companies, the amount of data about their customers in the future is not known, so they can store data in a local file on the computer. Become the savior of such a company and try to process the available data in the database.csv file to make it more business-friendly.

In the first stage, think about how to start working with the database.csv file and what data structure to choose for storing information about users.

If you open the database.csv file, for example, in your IDE, then the first lines will look like the screenshot below. This file has four columns: id, nickname, password, the consent to mailing. The last column may contain information on whether the user agrees to receive emails from the company. Each line of this file is an individual user. As you can see, some data may be missing; for example, the user with id = 2 has - instead of a nickname, which means that the nickname is lost.

Objectives

Your program must be able to:

  • Store the data from the database.csv file in the data structure of the programming language you use;

  • The output of your program should look like this: The user <name of last user> has "<consest to mailing of last user>" consent status for sending emails

<name of last user> is the user nickname from the last line of the database.csv file, <consest to mailing of last user> is the consent to mailing of the same user, the last one in the file.

Examples

Example 1: the last line contains information 24, Myjljczgg, g12q67lp, yes

The user Myjljczgg has "yes" consent status for sending emails

Example 2: the last line contains information 29, Fluplbokh, 1qsnvbzk, no

The user Fluplbokh has "no" consent status for sending emails
